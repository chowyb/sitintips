Unix Environment

The Unix environment is heavily used for development work during CS1010 and CS1020.
Here are a few tips to effectively use Unix for this purpose.

  • General
  • Vim
  • Commands

Terminal Layout

When using the Unix environment to code, it is advisable to use two terminals: one for writing the program, and one for compiling and running the program.
This reduces the time required to switch between writing and compiling/running, which may be done frequently when debugging code.

Vim Commands

If using Vim to write code, the following Vim commands may prove useful.
All commands must be run in Command mode (not Insert mode).

gg=G: Useful for auto-indenting code, and figuring out common errors such as a missed } symbol, or certain lines of code accidentally being placed inside/outside a loop or conditional.

Input/Output Redirection

In most cases, you will be provided with a set of sample input and output for your program.
Instead of manually typing in the input, it is possible to use a file to store your inputs (using <), and then redirect it to the program when running. This should be done when running the program.
Example for C: ./a.out < myinput
Example for Java: java MyProgram < myinput

This can also be combined with output redirection (using >) to redirect your output to another file.
Example for C: ./a.out < myinput > myoutput
Example for Java: java MyProgram < myinput > myoutput
Note that the above examples assume that the file myoutput does not exist. To overwrite a file if it exists, use either >! or >| (depending on environment settings) instead of >.

File Comparison

There are two different commands for comparing files, both of which use the same format:
cmp file1 file2 or diff file1 file2, where file1 and file2 are two files to be compared.
Combining this with output redirection, it makes comparing output much easier, as you can compare the output generated by your program (saved to a file) and the expected output.
Therefore, it becomes possible to do the following sequence of commands to compile, run and compare the output of your program:

Example for C:
gcc myprogram.c
./a.out < sampleinput.in > myoutput.out
cmp myoutput.out sampleoutput.out
 Example for Java:
javac MyProgram.java
java MyProgram < sampleinput.in > myoutput.out
cmp myoutput.out sampleoutput.out
